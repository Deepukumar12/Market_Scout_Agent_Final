ScoutIQ Prototype Walkthrough

A Comprehensive Guide to Demonstrating the Autonomous Market Intelligence Agent

Purpose: This document serves as both a Functional Prototype Specification and a Demonstration Script for presenting ScoutIQ to technical judges and stakeholders.

1. Prototype Scope (The MVP)

The current prototype demonstrates the core "Intelligence Loop":

  • Orchestration: Managing multiple asynchronous AI agents.
  • Acquisition: Real-time web searching and scraping (no mocked data).
  • Synthesis: Converting raw HTML into structured, strategic JSON reports.
  • Presentation: A glassmorphic, responsive Command Center dashboard.

Key Working Features

  • 🔐 Authentication: Secure Login/Register (JWT + OAuth2).
  • 📊 Command Center: Real-time dashboard with "Sector Radar" and "Live Signal Feed" (simulated).
  • 🤖 The Agent: Deep-dive capability (Input Company -> Get Report).
  • 📈 Visualizations: Three.js hero elements and Recharts data plotting.

2. Launch Instructions

To show this prototype, you must run the local development environment.

1 Prerequisites

Ensure .env in backend/ has valid GEMINI_API_KEY and SERPER_API_KEY.

2 Run Automation Script

Open Terminal in project root and run:

.\run.bat

This will spawn two windows: one for the FastAPI Backend (Port 8000) and one for the Vite Frontend (Port 5173).

3 Open Browser

Go to http://localhost:5173.

3. Demonstration Script (The "Happy Path")

Follow this exact sequence to demonstrate the platform's capabilities effectively.

Scene 1: The "Command Center"

Action: Log in and land on the Dashboard.

"This is the ScoutIQ Command Center. It gives us a macro-view of the competitive landscape. Notice the Glassmorphic Design—we prioritize a premium, futuristic aesthetic to match the advanced AI under the hood."
  • Point out KPI Cards (interactive time filters).
  • Point out 3D Visualizer (representing the Intelligence Network).

Scene 2: Initiating a Scan

Action: Click Initialize New Competitor.

"Let's see the agent in action. We want to track a new threat."

Input: Enter "Stripe" (or "Vercel") and click Start Scan.

Scene 3: The "Black Box" (Wait Time)

While the loading spinner is active (approx. 15-30s)...

"Right now, the system is executing a 5-step autonomous pipeline: Planning queries, Federated Search, Deep Scraping, Technical Filtering, and finally Synthesis."

Scene 4: The Reveal

Action: View the generated report.

"Here is the autonomous report. No human touched this. It found the new Payment Intent API and plotted it on our Sector Radar."
